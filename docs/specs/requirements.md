# 需求规范 (Requirements Derived from Design)

> **功能名称：** 校园导航系统前后端分离重构
> **来源设计：** `docs/specs/design.md`
> **版本：** v1.0
> **状态：** 草稿
> **最后更新：** 2026-03-30

---

## 1. 概述

本需求文档从已确认的技术设计（Django+DRF 后端、React+Electron 前端、PostgreSQL 数据库）中派生。所有需求均为现有 Flask 单体应用功能的 1:1 复现，不新增任何用户可感知能力。用户将获得与现有系统完全一致的校园导航体验，但运行在全新的前后端分离架构上。

---

## 2. 功能需求与验收标准

### REQ-001: POI 地图展示与分类筛选

**作为** 校园用户，**我希望** 在地图上看到所有校园地点并按分类筛选，**以便** 快速找到特定类型的地点。

#### 验收标准

- **AC-001.1: 加载全部 POI**
  - **GIVEN** 用户打开应用
  - **WHEN** 页面加载完成
  - **THEN** 地图上显示所有 POI 标记，每个标记颜色与其分类对应

- **AC-001.2: 分类筛选**
  - **GIVEN** 页面已加载，显示分类筛选按钮组
  - **WHEN** 用户点击某个分类按钮（如"食堂"）
  - **THEN** 地图上仅显示该分类的 POI 标记，其他标记隐藏

- **AC-001.3: 显示全部**
  - **GIVEN** 用户已选中某个分类筛选
  - **WHEN** 用户点击"全部"按钮
  - **THEN** 地图恢复显示所有分类的 POI 标记

- **AC-001.4: 分类颜色一致性**
  - **GIVEN** POI 数据已从 PostgreSQL 加载
  - **WHEN** 标记渲染到地图上
  - **THEN** 各分类颜色与现有系统一致：食堂(#FF4444)、教学楼(#4285F4)、宿舍(#34A853)、图书馆(#9C27B0)、运动场所(#FF9800)、行政(#607D8B)、医疗(#E91E63)

### REQ-002: POI 模糊搜索

**作为** 校园用户，**我希望** 通过关键词模糊搜索校园地点，**以便** 快速定位目标地点。

#### 验收标准

- **AC-002.1: 搜索返回结果**
  - **GIVEN** 用户在搜索框中输入关键词（如"食堂"）
  - **WHEN** 搜索请求发送至后端 `GET /api/pois/search/?q=食堂`
  - **THEN** 返回所有名称或描述与关键词模糊匹配度 >= 60 的 POI，按匹配度降序排列

- **AC-002.2: 搜索结果展示**
  - **GIVEN** 搜索返回了结果列表
  - **WHEN** 结果渲染到下拉列表中
  - **THEN** 每条结果显示 POI 名称和分类

- **AC-002.3: 点击搜索结果定位**
  - **GIVEN** 搜索下拉列表中显示了结果
  - **WHEN** 用户点击某条结果
  - **THEN** 地图中心移动到该 POI 位置，并打开其信息窗口

- **AC-002.4: 空关键词处理**
  - **GIVEN** 搜索框为空
  - **WHEN** 触发搜索
  - **THEN** 返回空数组，不显示下拉列表

### REQ-003: 两点步行路径规划

**作为** 校园用户，**我希望** 规划从一个地点到另一个地点的步行路线，**以便** 了解路线、距离和预计时间。

#### 验收标准

- **AC-003.1: 路径规划成功**
  - **GIVEN** 用户选择了起点和终点 POI
  - **WHEN** 发送路径规划请求 `GET /api/routing/walk/`
  - **THEN** 返回路线坐标数组、距离（米）和时间（分钟），地图上绘制蓝色路线

- **AC-003.2: 路径信息展示**
  - **GIVEN** 路径规划成功返回
  - **WHEN** 结果渲染到界面
  - **THEN** 显示总距离（米）和预计步行时间（分钟）

- **AC-003.3: 参数错误处理**
  - **GIVEN** 请求缺少必要坐标参数或参数格式错误
  - **WHEN** 后端接收到请求
  - **THEN** 返回 HTTP 400，body 为 `{"error": "参数错误"}`

### REQ-004: 多点路径规划

**作为** 校园用户，**我希望** 规划经过多个地点的步行路线，**以便** 一次性访问多个目的地。

#### 验收标准

- **AC-004.1: 多点路径规划成功**
  - **GIVEN** 用户添加了 2 个或以上途经点
  - **WHEN** 发送多点路径规划请求 `POST /api/routing/multi/`
  - **THEN** 返回合并后的路线坐标、总距离和总时间，地图上绘制完整路线

- **AC-004.2: 动态添加途经点**
  - **GIVEN** 路径规划面板已打开
  - **WHEN** 用户点击"添加途经点"
  - **THEN** 新增一个 POI 选择下拉框

- **AC-004.3: 途经点不足处理**
  - **GIVEN** 用户提交的途经点少于 2 个
  - **WHEN** 后端接收到请求
  - **THEN** 返回 HTTP 400，body 为 `{"error": "至少需要两个途经点"}`

### REQ-005: 地图交互

**作为** 校园用户，**我希望** 与地图上的标记和校区边界进行交互，**以便** 获取地点详细信息。

#### 验收标准

- **AC-005.1: 校区边界绘制**
  - **GIVEN** 地图配置已从 `GET /api/config/map/` 加载
  - **WHEN** 地图初始化完成
  - **THEN** 校区边界以多边形形式绘制在地图上（22 个坐标点）

- **AC-005.2: POI 标记点击**
  - **GIVEN** 地图上显示了 POI 标记
  - **WHEN** 用户点击某个标记
  - **THEN** 弹出信息窗口，显示 POI 名称、分类和描述

- **AC-005.3: 地图中心与缩放**
  - **GIVEN** 应用启动
  - **WHEN** 地图初始化
  - **THEN** 地图中心为 [103.988471, 30.581856]，缩放级别为 16

### REQ-006: Electron 桌面应用

**作为** 校园用户，**我希望** 通过桌面应用使用校园导航系统，**以便** 获得原生应用体验。

#### 验收标准

- **AC-006.1: 应用启动**
  - **GIVEN** Electron 应用已安装
  - **WHEN** 用户双击启动
  - **THEN** 打开 1200x800 窗口，标题为"校园导航系统 - 成信大航空港校区"，加载完整导航界面

- **AC-006.2: 窗口约束**
  - **GIVEN** 应用窗口已打开
  - **WHEN** 用户尝试缩小窗口
  - **THEN** 窗口最小尺寸不低于 900x600

- **AC-006.3: 关闭行为**
  - **GIVEN** 应用正在运行
  - **WHEN** 用户关闭窗口
  - **THEN** 应用完全退出

## 3. 非功能性需求

| ID | 类别 | 描述 | 来源设计约束 |
|:---|:---|:---|:---|
| NFR-001 | 性能 | POI 列表接口响应时间 < 200ms | 数据量 52 条，PostgreSQL 查询 |
| NFR-002 | 性能 | 模糊搜索接口响应时间 < 500ms | rapidfuzz 全表扫描 52 条 |
| NFR-003 | 性能 | 路径规划响应时间取决于高德 API，后端转发延迟 < 100ms | 高德 Web 服务 API 约束 |
| NFR-004 | 兼容性 | Electron 应用支持 Windows 10/11 | Electron + React 构建 |
| NFR-005 | 可维护性 | 前后端代码完全分离，通过 REST API 通信 | Monorepo 结构设计 |
| NFR-006 | 数据一致性 | PostgreSQL 中的 POI 数据与原 poi.json 完全一致 | 数据迁移脚本验证 |

## 5. 约束、假设与超出范围

### 约束

- 所有功能必须与现有 Flask 系统行为一致
- 模糊搜索必须使用 rapidfuzz partial_ratio，阈值 60
- 高德地图 API 密钥不可更换

### 假设

- PostgreSQL 数据库在本地可用
- Node.js 18+ 和 Python 3.10+ 已安装
- 高德 API 密钥在开发期间有效

### 超出范围

- 用户认证与权限管理
- POI 数据增删改管理界面
- 移动端适配
- 离线地图缓存
- CI/CD 与容器化部署
